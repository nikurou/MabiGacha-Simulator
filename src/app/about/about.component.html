About this Project

MabiGacha is a tool to simulate opening gachapons from the Korean MMORPG title, Mabinogi. MabiGacha first first went live in 2019. It served more as a personal/practice project for two recent Comp Sci college grads, however we never shared it with the community at the time.

A new gachapon has been released, why doesn't MabiGacha support it?

I no longer play this game, so unless someone notifies me, I am not really aware of updates and releases and new gachas. So give me a ping if something new happens that requires changes from us, and I'll upload the new gacha asap!

How do we know this is accurate?

South Korea's Fair Trade Commission has required video game developers to accurately disclose the odds of rewards offered in loot boxes. As Mabinogi is published in Korea, we have access to those numbers. So for every pull you do, we are using the exact same rates as the game though the in-game gacha algorithm may be different, this is a close approximation!

What algorithm do you employ to emulate pulls?

The algorithm being used is called weighted sampling where an array is populated with items as weighted by their probabilities, and then one is drawn from the pool at random. For example, if item A has a 30% probability of being drawn, we populate 30 indexes of the array with item A. And so on with Item B, C,.... Then we randomly draw from the array.

How is the total NX spent calculated?

Currently, we have the hardcoded assumption that all gachas are sold in quantities of 1, 11, and 45 with each option costing 1500 NX, 15000 NX, 57500 NX respectively. When you select the option to gach in those quantities, we will use those prices. When you use our 'specific' gacha, we make the assumption that you bought your gachapons in the most efficient manner and calculate a price.

For example, if a user needed 50 attempts to pull a specific item, X, we will assume the user purchased one of the 45x Bundle and 5 1x Bundles. Resulting in 57500 + (5 * 1500) = 65,000 NX
